# Документация
Это приложение предоставляет инструмент для получения текста статьи из URl.
*Приложение ориентировано на Windows OS*

## Установка
Склонируйте этот репозиторий себе на устройство и войдите в папку ```ArticleParser```

Чтобы установить зависимости используйте эту команду:
```commandline
pip install -r requirements.txt
```

## Использование
Чтобы использовать приложение введите в терминал эту команду, указав нужный URL во флаге --url:
```commandline
ArticleParser.py --url=https://yoursite/
```

Также вы можете настроить формат выходного файла с помощью файла ```settings.txt```.
В этом файле вы можете отметить параметры как "yes", если хотите вывести их в выходной файл, 
или ввести число в параметре ``Line length limit``.

Поcле выполнения программы выходной каталог с текстом статьи будет создан автоматически в текущем каталоге.

## Описание алгоритма работы 

Алгоритм состоит в последовательном вызове методов класса ```ArticleParser```.

```AP.get_args() ```
Получает URL страницы от пользователя через команду терминала с использованием обязательного флага --url.
Получает настройки параметров форматирования из файла settings.txt

```AP.getter()```
Получает содержимое страницы  из url. При необходимости добавляет в выходной текст url и заголовок статьи.

```AP.formatter()```
Форматирует полученный сырой текст статьи под нужный формат.
Для каждого абзаца текста находит в нем гиперссылки, если ссылка относительна (ссылка на другую страницу этого же сайта), то добавляет к ней протокол и доменное 
имя сайта, чтобы сделать ссылку абсолютной. 
Добавляет слова к тексту по одному слову, каждый раз проверяя длину текущей строки. Если длина превышает лимит, то
переводит текущее на следующую строку. В случае если слово (например, длинная ссылка) больше заданного лимита строки, то 
слово разделяется на части и переносится по частям на следующую строку.

```AP.writer()```
Печатает отформатированный текст в файл. 
Формирует путь к выходному файлу из url. Создает новые папки в текущей директории, 
если их не существует. Название выходного файла формируется по последней части гкд отделенной слешем.
Выводит сообщение о записи файла и путь его местонахождения.

## Дальнейшее развитие
Программа имеет потенциал к развитию:
1. Возможно добавление вывода в другой текстовый формат, например, .md
2. Возможно добавление картинок в текст, имеющих прямое отношению к статье.
3. Возможно добавление других параметров в файле ```settings.txt```
4. Возможно добавление асинхронной обработки текста.
